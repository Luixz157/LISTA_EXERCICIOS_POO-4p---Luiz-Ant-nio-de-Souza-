-------------------- Conceito - SOLID - POO --------------------

src -> pt_br -> srp _ ocp _ lsp _ isp _ dip

1 - SRP = Single responsibility principle ->
Cada classe deve ter uma única responsabilidade ou motivo para mudar. Isso significa que
 uma classe deve fazer apenas uma coisa e ser especializada em uma única tarefa ou funcionalidade.

2 - OCP = Open-Closed principle ->
As classes devem ser abertas para extensão, mas fechadas para modificação. Ou seja, é possível adicionar
 novos comportamentos ou funcionalidades sem alterar o código existente, minimizando os riscos de introduzir erros.

3 - LSP = Liskov substitution principle ->
As subclasses devem poder substituir suas classes base sem alterar o comportamento correto do programa.
 Em outras palavras, um objeto da classe filha deve poder ser usado no lugar de um objeto da classe pai sem problemas.

4 - ISP = Interface Segregation principle ->
 Em vez de criar interfaces grandes e abrangentes, crie interfaces menores e específicas para o cliente.
  As classes devem ser obrigadas a implementar apenas os métodos que realmente utilizam, evitando dependências desnecessárias.

5 - DIP = Dependency Invesion Principle ->
Módulos de alto nível não devem depender de módulos de baixo nível. Ambos devem depender de 
abstrações (interfaces). Isso promove um design desacoplado e facilita a manutenção e a evolução do código.

verificar se está disponível.

informações da pessoa.
